DEVOPS CHALLENGE

Práctica de Métricas de Software

Contexto

La empresa tecnológica Marea Café Systems necesita modernizar su plataforma de software. Actualmente los despliegues son manuales, existen errores frecuentes y no se mide el rendimiento del proceso. Cada equipo será responsable de un módulo del sistema y deberá aplicar prácticas DevOps.

Objetivo

Implementar un flujo DevOps funcional y medir el desempeño del equipo utilizando métricas oficiales DORA.

Equipo (TSP) - Ventas API

Equipo (XP) - Inventario API
Equipo (DAS) - Clientes API

Flujo DevOps

  • F1 - Planificación del sprint
  • F2 - Desarrollo del módulo asignado
  • F3 - Commit y versionado en Git
  • F4 - Build automático
  • F5 - Testing del sistema
  • F6 - Deploy local
  • F7 - Monitoreo
  • F8 - Medición de métricas DORA

Métricas DORA a medir

Deployment Frequency

Cuántas veces el equipo despliega cambios exitosamente.

Change Failure Rate

Porcentaje de despliegues que fallan o generan errores.

Tasa de Errores por Modificación

Cantidad de bugs detectados después de cada cambio.

Tiempo de Recuperación

Tiempo necesario para restaurar el sistema tras un despliegue fallido.

Entregables

  • Código funcional del módulo
  • Reporte de métricas DORA
